# MixCut — 产品规划文档

> AI 驱动的广告视频混剪工具 | Mac 原生桌面应用

---

## 一、产品定位

### 1.1 一句话描述

**MixCut 是一款面向广告投放团队的 Mac 桌面工具，用户导入几条人工筛选的广告素材视频，AI 自动按语义切分镜头、标注类型，然后智能排列组合生成多条全新的混剪广告视频。**

### 1.2 解决的问题

| 痛点 | 现状 | MixCut 方案 |
|------|------|------------|
| 广告视频制作成本高 | 每条视频需要剪辑师手动操作，耗时 2-4 小时 | AI 自动切分+组合，5 分钟出多条成品 |
| 素材利用率低 | 一条素材视频只产出一条广告 | 多条素材交叉混剪，一组素材可产出 10+ 条差异化广告 |
| A/B 测试素材不足 | 手动制作变体成本太高，测试覆盖不够 | 批量生成差异化方案，自动覆盖不同风格和受众 |
| 投放人员不懂剪辑 | 依赖设计团队排期，响应慢 | 无需剪辑技能，导入-生成-导出三步完成 |

### 1.3 目标用户

- **广告投放团队**：电商、品牌方的投放优化师
- **MCN/代运营公司**：批量产出短视频广告素材
- **品牌方市场部**：快速制作多版本广告进行 A/B 测试
- **短视频内容工作室**：提高视频产出效率

### 1.4 竞品差异

| 对比维度 | MixCut | 剪映/CapCut | 传统 NLE (Premiere) |
|---------|--------|------------|-------------------|
| 核心能力 | AI 语义混剪 | 通用剪辑 | 专业剪辑 |
| 上手门槛 | 零剪辑基础 | 需要基础剪辑技能 | 需要专业技能 |
| 产出效率 | 5分钟/10条 | 2小时/1条 | 4小时/1条 |
| AI 深度 | 语义理解+智能编排 | 模板套用 | 无 |
| 运行方式 | 本地离线 | 云端/本地 | 本地 |

### 2.2 核心流水线（用户操作流程）

```
用户操作          系统处理                     产出
────────────────────────────────────────────────────────

1. 创建项目       →                            → 空项目
   (命名)

2. 导入视频       → 拷贝到应用目录              → 视频文件列表
   (拖拽/选择)    → 提取元数据(时长/分辨率/帧率)
                  → 生成缩略图

3. [等待] AI分析  → 镜头检测(FFmpeg)  ─┐        → 分镜素材库
                  → 语音识别(Whisper)  ─┤ 并行
                                       ↓
                  → AI语义分析
                  → 边界优化(吸附算法)
                  → 生成分镜列表+标签+评分

4. 浏览分镜       → (用户可筛选/搜索/修改标签)   → 确认分镜质量
   (可选操作)

5. 生成方案       → Step1: AI生成策略            → N个混剪方案
   (设置数量)     → Step2: AI选择分镜组合

6. 调整方案       → (用户拖拽排序/替换分镜)       → 最终方案
   (可选操作)     → 预览播放

7. 导出           → FFmpeg裁切+拼接              → MP4文件
   (设置参数)     → 进度显示
```

### 2.3 详细功能说明

#### 功能①：素材导入

| 子功能 | 说明 |
|--------|------|
| 拖拽导入 | 直接拖拽视频文件到导入区域 |
| 文件选择器 | NSOpenPanel 支持多选 |
| 支持格式 | MP4, MOV, AVI, M4V |
| 元数据提取 | 自动读取：时长、分辨率、帧率、编码格式 |
| 缩略图 | 自动在视频第 1 秒处截图 |
| 文件管理 | 拷贝到应用沙盒目录，原始文件不受影响 |
| **素材去重** | **导入时自动检测重复视频（基于文件名+文件大小），跳过已存在的视频并提示用户** |
| **预览比例** | **视频缩略图/预览使用视频实际宽高比（非固定 16:9），支持横屏、竖屏、方形等多种比例** |
| **处理状态可视化** | **视频卡片实时显示当前处理步骤（步骤 1/3 镜头检测、2/3 语音识别、3/3 AI 分析），处理中显示半透明遮罩+加载动画，失败显示红色遮罩** |
| **任务取消** | **删除视频时自动取消该视频的等待中/进行中的分析任务，不影响其他视频的处理** |

#### 功能②：AI 智能分析（核心差异化能力）

> **核心架构：本地数据驱动 + AI 语义决策**
>
> 不直接将视频传给 AI（视频理解按 1fps 抽帧，切割精度不足）。
> 所有视觉/音频信号由本地 FFmpeg 精确提取，转化为结构化数据后传给 AI，AI 仅做语义层面的切分决策。

**本地分析阶段（FFmpeg 精确提取）**

| 分析项 | 工具 | 输出 | 用途 |
|--------|------|------|------|
| 画面切换检测 | FFmpeg `select='gt(scene,0.3)',showinfo` | 时间点 + 置信度 | 确定镜头边界，避免画面混入 |
| 音频静音检测 | FFmpeg `silencedetect=noise=-30dB:d=0.3` | 静音段起止时间 | 找到最自然的切割位置（说话间停顿） |
| I-frame 关键帧提取 | FFmpeg `select='eq(pict_type,I)',showinfo` | I-frame 时间戳列表 | 确保在完整帧处切割，copy 模式不花屏 |
| 语音识别（ASR） | Whisper（本地执行，支持 Python openai-whisper / whisper.cpp） | 逐句文本 + 精确起止时间戳 | 保证台词完整性，不在句子中间切断 |

**AI 语义分析（千问/MiniMax，OpenAI 兼容 API）**
- **不接收视频画面**，仅接收上述结构化数据
- 输入：ASR 逐句转录（含时间戳）+ 画面切换点 + 静音段 + I-frame 分布 + 视频元信息
- AI 综合所有信号判断最优切割位置，确保：语义独立、台词完整、画面干净
- 输出：每个分镜的起止时间、台词、语义类型、位置类型、质量评分

**语义类型体系（11 种）**

| 类型 | 说明 | 典型时长 |
|------|------|---------|
| 噱头引入 (Hook) | 开场吸引注意力 | 2-10 秒 |
| 痛点 (Pain Point) | 场景化描述用户问题 | 5-15 秒 |
| 产品方案 (Solution) | 产品功能/成分介绍 | 8-20 秒 |
| 效果展示 (Results) | 使用前后对比 | 5-12 秒 |
| 信任背书 (Social Proof) | 用户评价/品牌背书 | 5-12 秒 |
| 价格对比 (Price Anchor) | 价格锚点设置 | 3-8 秒 |
| 活动福利 (Promotion) | 限时优惠/赠品 | 5-20 秒 |
| 行动号召 (CTA) | 购买指令 | 3-10 秒 |
| 产品定位 | 适用人群/场景 | 3-8 秒 |
| 产品使用教育 | 使用方法 | 5-15 秒 |
| 过渡 | 承上启下 | 1-3 秒 |

**位置类型（3 种）**

| 位置 | 规则 |
|------|------|
| 开头 | 承担"吸引注意力"功能，混剪时放在新视频前 20% |
| 中间 | 核心内容，可灵活排列 |
| 结尾 | 承担"驱动转化"功能，混剪时放在新视频后 20% |

**质量评分（0-10 分）**
- 数据表现（40%）：点击率、留存率
- 内容质量（30%）：语义完整性、信息密度
- 混剪价值（30%）：时长适中、可独立理解

**四阶段边界优化算法**

AI 输出的切点经过四阶段精调，解决「画面切割不干净」和「台词没说完」两大问题：

| 阶段 | 策略 | 搜索范围 | 解决的问题 |
|------|------|---------|-----------|
| ① ASR 句子结束吸附 ⭐ | 切点吸附到最近的句子 endTime | ±1.5s | 台词没说完 |
| ② 场景切换对齐 | 对齐到最近的画面切换点 | ±0.5s | 画面切割不干净（镜头混入） |
| ③ 静音段吸附 | 移到最近的停顿/静音段中点 | ±0.8s | 切在说话中间不自然 |
| ④ I-frame 对齐 | 微调到最近的 I-frame | ±0.15s | copy 模式花屏 |

**安全约束**
- 最大边界移动距离：2.0 秒（防止过度偏移）
- 最小片段时长：1.0 秒（防止碎片化）
- 边界严格递增 + 不超过视频时长

#### 功能③：方案生成（两步架构）

**Step 1 — 策略层**
AI 生成方案策略，不涉及具体分镜选择：
- 方案名称和风格
- 目标受众定位
- 叙事结构设计
- 目标时长范围

**10 种核心说服策略**

| 策略 | 适用场景 | 目标时长 |
|------|---------|---------|
| 完整信息路径型 | 理性用户 | 85-110 秒 |
| 快速冲击型 | 信息流场景 | 50-70 秒 |
| 痛点堆叠型 | 痛点明显的产品 | 70-85 秒 |
| 价格爆破型 | 促销期 | 60-80 秒 |
| 反套路型 | 广告免疫人群 | 70-85 秒 |
| 专业教育型 | 需要指导的用户 | 70-90 秒 |
| 信任堆叠型 | 高信任门槛产品 | 75-95 秒 |
| 场景营销型 | 季节性产品 | 80-100 秒 |
| 情感共鸣型 | 母婴/礼品 | 75-95 秒 |
| 综合型 | 通用场景 | 80-100 秒 |

**10 种广告风格**

| 风格 | 镜头时长 | 适用品类 |
|------|---------|---------|
| 快闪 | 1-2 秒/镜头 | 科技、运动 |
| 故事叙事 | 5-8 秒/镜头 | 高端产品 |
| 简约直给 | 3-5 秒/镜头 | 日用品 |
| 对比展示 | 4-6 秒/镜头 | 美妆护肤 |
| 教程演示 | 6-10 秒/镜头 | 厨房家电 |
| UGC | 3-6 秒/镜头 | 口播种草 |
| 情感共鸣 | 8-12 秒/镜头 | 母婴宠物 |
| 炫技酷炫 | 2-4 秒/镜头 | 潮牌游戏 |
| 促销紧迫 | 2-3 秒/镜头 | 限时促销 |
| 高端品质 | 6-10 秒/镜头 | 奢侈品 |

**Step 2 — 执行层**
基于策略，AI 选择具体分镜并排列：
- 从分镜素材库中筛选符合策略的片段
- 按叙事结构排列（Hook → Body → Climax → CTA）
- 遵守位置约束（开头片段不放结尾，结尾片段不放开头）
- 语义连贯性检查（避免逻辑跳跃）
- 差异化校验（与已有方案对比，确保差异度 ≥ 75 分）

**差异化维度**

| 维度 | 权重 | 说明 |
|------|------|------|
| 策略差异 | 30% | 不同的核心说服策略 |
| 开场差异 | 20% | 不同的开场方式 |
| 时长差异 | 15% | 覆盖不同时长区间 |
| 片段组合差异 | 20% | 重复率 < 60% |
| 受众差异 | 15% | 不同的目标受众 |

## 三、技术架构

### 3.1 技术栈

| 组件 | 选型 | 理由 |
|------|------|------|
| 框架 | SwiftUI | macOS 原生体验，声明式 UI |
| 数据 | SwiftData | Apple 官方 ORM，无需引入第三方 |
| 视频处理 | FFmpeg (bundled) | 业界标准，功能最全 |
| 语音识别 | whisper.cpp / WhisperKit | 本地执行，保护数据隐私 |
| AI 分析 | 千问 / MiniMax（OpenAI 兼容 API） | 多提供商支持，成本可控 |
| 密钥存储 | UserDefaults | 开发期简化存储（避免钥匙串弹窗） |

### 3.2 架构分层

```
┌─────────────────────────────────────────────────────┐
│              Presentation (SwiftUI + MVVM)           │
│  ProjectVM │ ImportVM │ SegmentLibraryVM │ SchemeVM  │
├─────────────────────────────────────────────────────┤
│                   Service Layer                      │
│  SceneDetection │ ASR │ AIAnalysis │ SchemeGen │ Export│
├─────────────────────────────────────────────────────┤
│                   Data Layer (SwiftData)              │
│  Project │ ProjectVideo │ Video │ Segment │ MixScheme │ SchemeSegment│
├─────────────────────────────────────────────────────┤
│                   Infrastructure                     │
│  FFmpeg (bundled) │ whisper.cpp │ 千问/MiniMax API │ AVKit │
└─────────────────────────────────────────────────────┘
```

### 3.3 数据模型

```
Project (项目)  *──*  Video (视频，全局共享)
│                     ├── id, name, localPath, contentHash
│  通过 ProjectVideo  ├── duration, width, height, fps
│  多对多关联          ├── transcript, asrWords (字级时间戳)
│                     └── segments[] ──→ Segment (分镜，全局共享)
│                                        ├── segmentIndex, startTime, endTime
│                                        ├── text (台词)
│                                        ├── semanticTypes (11种，支持多个)
│                                        ├── positionType (3种，单个)
│                                        ├── qualityScore, keywords
│                                        └── thumbnailPath
├── schemes[] ──→ MixScheme (方案)
│                 ├── name, style, description
│                 ├── targetAudience, narrativeStructure
│                 ├── strategyReasoning, differentiation
│                 └── schemeSegments[] ──→ SchemeSegment (有序关联)
│                                         ├── position (排列顺序)
│                                         ├── reasoning (选择理由)
│                                         └── segment → Segment
└── projectVideos[] ──→ ProjectVideo (多对多中间表)
                        ├── project → Project
                        ├── video → Video
                        └── addedAt (加入时间)

核心设计：
- 同一视频文件（SHA-256 哈希相同）全局只有一个 Video 实体和一组 Segment
- 多个项目通过 ProjectVideo 共享引用同一份数据
- 在任何项目中修改分镜，所有项目立即同步
- 导入已分析的视频秒级完成（直接创建关联，不重复处理）
- 删除项目时，仅当视频无其他项目引用才清理磁盘文件
```

### 3.4 核心数据流

```
导入视频 ──→ AVFoundation 提取元数据 + FFmpeg 生成缩略图
         │
         ├──→ 本地视频分析（SceneDetectionService.analyzeLocally）
         │    ├── FFmpeg 画面切换检测 → [SceneBoundary]
         │    ├── FFmpeg 静音段检测   → [SilencePeriod]
         │    └── FFmpeg I-frame 提取 → [Double]（时间戳）
         │    = VideoLocalAnalysis（完整本地分析结果）
         │
         └──→ Whisper ASR 语音识别 → TranscriptionResult（逐句+时间戳）
                                    │
                                    ↓
              AI 语义分析（仅接收结构化文本数据，不接收视频）
              输入：ASR 句子 + 画面切换 + 静音段 + I-frame 分布
              输出：AISegmentationResult（分镜列表+类型+评分）
                                    │
                                    ↓
              四阶段边界优化（BoundaryOptimizerService）
              ① 句子吸附 → ② 场景对齐 → ③ 静音吸附 → ④ I-frame 对齐
                                    │
                                    ↓
                   分镜素材库 (Segment[])
                                    │
                                    ↓
                   方案生成 Step1: 策略 ──→ Step2: 组合
                                    │
                                    ↓
                   FFmpeg 裁切 + 拼接 ──→ MP4 导出
```

## 七、关键技术难点

| 难点 | 解决方案 | 风险等级 |
|------|---------|---------|
| SwiftData 宏只能在 Xcode 编译 | 用 Xcode 打开 Package.swift 编译 | 低 |
| FFmpeg/Whisper 打包到 App Bundle | bundle_deps.py 复制二进制+dylib，修复 @loader_path，ad-hoc 签名 | 中（已解决） |
| AI API 输出不稳定 | JSON Schema 约束 + 重试 + 解码验证 | 中 |
| 视频处理性能 | 镜头检测+ASR 并行(TaskGroup)，GPU 加速 | 低 |
| 沙盒文件访问 | Security-Scoped Bookmarks | 低 |
| 大量分镜 UI 卡顿 | LazyVGrid + NSCache 缩略图缓存 | 低 |
