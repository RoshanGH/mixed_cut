# 视频重组混剪方案生成提示词模板

> 本文档提供了高质量视频混剪方案生成的完整提示词模板，基于实际验证的成功案例（10个方案，100个视频，100%可用率）。

---

## 📋 提示词完整模板

### 基础版本（单个方案生成）

```
# 任务：视频混剪方案生成

你是一个专业的视频混剪策划师。请基于提供的语义片段，设计一个高质量的混剪方案。

## 输入数据

### 1. 所有可用的语义片段
{all_segments}

每个片段包含：
- segment_id: 片段唯一标识
- source_video_id: 来源视频
- type: 片段类型（噱头引入/痛点/产品方案/效果展示/信任背书/价格对比/活动福利/行动号召等）
- position: 原始位置（开头/中间/结尾）
- text: 完整文本内容
- duration: 时长（秒）
- start_time / end_time: 在源视频中的精确时间点
- data_quality.score: 质量评分（0-10）

### 2. 混剪原则参考
{recombination_principles}

### 3. 广告风格定义
{ad_styles}

### 4. 方案生成参数
- 目标时长范围：{min_duration} - {max_duration} 秒
- 方案编号：第 {scheme_number} 个方案
- 已生成方案：{existing_schemes}（需要与它们有明显差异）

## 方案设计的六步决策流程

### 步骤1️⃣：确定方案策略（核心定位）

你需要为这个方案选择一个独特的策略定位，从以下维度思考：

#### A. 目标受众类型（选择一种）
```
理性决策型
- 特征：需要完整信息、专业背书、成分讲解
- 时长策略：85-110秒（充分说服）
- 节奏：慢速，每段8-15秒
- 示例：深度种草型、功能教育型

冲动决策型
- 特征：快速刺激、价格敏感、紧迫感驱动
- 时长策略：50-75秒（快速转化）
- 节奏：快速，每段5-8秒
- 示例：快节奏冲击型、价格爆破型

特定人群型
- 特征：针对特定场景、心理、时间点
- 时长策略：70-90秒（适中）
- 节奏：中等，每段7-12秒
- 示例：秋冬场景营销、女性导向型、反套路型

通用受众型
- 特征：多策略融合、覆盖面广
- 时长策略：80-100秒
- 节奏：变化，片段多样
- 示例：综合型、多角度攻击型
```

#### B. 核心说服策略（选择一种主导策略）
```
1. 完整信息路径型
   叙事结构：痛点 → 产品方案 → 效果展示 → 信任背书 → 价格对比 → 活动福利 → CTA
   适用：理性用户，需要充分说服
   时长：85-110秒

2. 快速冲击型
   叙事结构：噱头引入 → 快速痛点 → 产品亮点 → 价格/福利 → CTA
   适用：信息流快速滑动场景
   时长：50-70秒

3. 痛点堆叠型
   叙事结构：多维度痛点展示 → 产品方案 → 效果证明 → CTA
   适用：痛点明显的产品
   时长：70-85秒

4. 价格爆破型
   叙事结构：价格冲击 → 产品简介 → 福利堆叠 → 紧迫CTA
   适用：促销期、价格敏感用户
   时长：60-80秒

5. 反套路型
   叙事结构：反向开场 → 承认缺点 → 真实效果 → 理性劝退 → 福利
   适用：广告免疫的年轻人
   时长：70-85秒

6. 专业教育型
   叙事结构：问题分析 → 成分讲解 → 使用教育 → 效果展示 → CTA
   适用：需要选择指导的用户
   时长：70-90秒

7. 信任堆叠型
   叙事结构：多重背书 → 产品方案 → 用户见证 → 效果对比 → CTA
   适用：信任门槛高的产品
   时长：75-95秒

8. 场景营销型
   叙事结构：场景引入 → 痛点关联 → 产品方案 → 效果展示 → 福利 → CTA
   适用：季节性、时间点相关的产品
   时长：80-100秒

9. 情感共鸣型
   叙事结构：情感唤醒 → 心理共鸣 → 产品解决 → 效果展示 → CTA
   适用：女性、母婴、礼品类
   时长：75-95秒

10. 综合型
    叙事结构：多策略融合，灵活组合
    适用：通用场景
    时长：80-100秒
```

#### C. 广告风格选择（从 ad_styles 中选择）
```
快闪风格 - 1-2秒/镜头，信息流快速转化
故事叙事风格 - 5-8秒/镜头，高端品牌认知
简约直给风格 - 3-5秒/镜头，日用品种草
对比展示风格 - 4-6秒/镜头，效果型产品
教程演示风格 - 6-10秒/镜头，美妆护肤教程
UGC风格 - 3-6秒/镜头，真实体验口播
情感共鸣风格 - 8-12秒/镜头，母婴礼品
促销紧迫风格 - 2-3秒/镜头，限时秒杀
高端品质风格 - 6-10秒/镜头，奢侈品
```

**决策输出（步骤1）**：
```
我选择的策略是：{策略类型}
目标受众：{受众描述}
核心说服策略：{说服策略}
广告风格：{风格名称}
目标时长：{min_duration}-{max_duration}秒
```

### 步骤2️⃣：数据驱动的片段筛选

基于步骤1确定的策略，从所有片段中筛选符合条件的候选片段。

#### 筛选规则

**规则1：质量优先**
```
必选片段：data_quality.score >= 8.5
优选片段：data_quality.score >= 8.0
可选片段：data_quality.score >= 7.5
避免片段：data_quality.score < 7.5
```

**规则2：类型匹配**

根据步骤1的策略，确定需要的片段类型组合：

```
示例1：深度种草型需要
✓ 必须：痛点 × 1-2，产品方案 × 1-2，效果展示 × 1，CTA × 1
✓ 推荐：信任背书 × 1，价格对比 × 1，活动福利 × 1
✓ 可选：噱头引入 × 1

示例2：价格爆破型需要
✓ 必须：价格对比 × 1-2，活动福利 × 1，CTA × 1
✓ 推荐：产品方案 × 1，噱头引入 × 1
✓ 可选：效果展示 × 1

示例3：反套路型需要
✓ 必须：噱头引入（反套路型）× 1，效果展示 × 1，CTA × 1
✓ 推荐：痛点 × 1，产品方案 × 1
✓ 可选：活动福利 × 1
```

**规则3：位置约束遵守 ⭐️⭐️⭐️（非常重要）**

```
开头片段（position="开头"）：
✓ 优先用在新视频的开头位置（前20%时长）
✓ 可作为开场钩子、第一印象
✗ 避免用在新视频的结尾
理由：开头片段有"开场感"，放结尾会很突兀

中间片段（position="中间"）：
✓ 可以灵活排列，只要语义连贯
✓ 可用在新视频的任何位置
✓ 根据叙事需要重新排序组合

结尾片段（position="结尾"）：
✓ 优先用在新视频的结尾位置（后20%时长）
✓ 可作为总结、行动号召、紧迫感
✗ 避免用在新视频的开头
理由：结尾片段有"收尾感"，放开头会逻辑混乱
```

**规则4：时长计算**
```
目标时长范围：{min_duration} - {max_duration} 秒

片段数量建议：
- 50-70秒：选择 5-7 个片段
- 70-90秒：选择 7-9 个片段
- 90-110秒：选择 8-10 个片段

平均片段时长 = 目标时长 / 片段数量
- 快节奏：6-8秒/片段
- 中等：8-12秒/片段
- 慢节奏：12-15秒/片段
```

**决策输出（步骤2）**：
```
候选片段池：
[列出所有符合条件的片段ID、类型、时长、评分]

初步选择：
开头区域候选：[seg_XXX, seg_YYY]
中间区域候选：[seg_AAA, seg_BBB, seg_CCC, ...]
结尾区域候选：[seg_ZZZ]
```

### 步骤3️⃣：语义匹配与逻辑连贯性检查

从候选池中选择最终片段，并排序。

#### 语义匹配原则

**✅ 允许的组合（自然的逻辑流）**
```
痛点 → 产品方案          （问题 → 解决方案）
噱头引入 → 痛点           （吸引 → 共鸣）
产品方案 → 效果展示       （功能 → 结果）
效果展示 → 信任背书       （效果 → 证明）
信任背书 → 价格对比       （信任 → 价格）
价格对比 → 活动福利       （价格 → 超值）
活动福利 → 行动号召       （促销 → 转化）
产品方案 → 产品方案       （功能展开）
痛点 → 痛点               （多维痛点）
```

**❌ 避免的组合（逻辑倒置或跳跃）**
```
行动号召 → 痛点           （已经CTA了又回到问题）
产品方案 → 噱头引入       （已经进正题了又回开场）
活动福利 → 产品使用教育   （紧迫感后又开始教育）
价格对比 → 痛点           （已经谈价格了又回问题）
效果展示 → 产品方案       （已经展示结果了才讲功能）
```

#### 片段排序策略

**策略A：标准递进型**
```

### 步骤4️⃣：位置约束验证（关键检查）

对步骤3的排序结果进行位置约束检查。

#### 检查清单

**检查1：开头片段的使用**
```
新视频的前20%位置（0 ~ {总时长*0.2}秒）：
✓ 只能使用 position="开头" 或 position="中间" 的片段
✗ 绝对不能使用 position="结尾" 的片段

示例（假设总时长80秒）：
前16秒（0-16s）的片段必须是：
- seg_001 (position="开头") ✓
- seg_002 (position="开头") ✓
- seg_003 (position="中间") ✓

错误示例：
- seg_008 (position="结尾", type="行动号召") ✗ 不能放开头
```

**检查2：结尾片段的使用**
```
新视频的后20%位置（{总时长*0.8} ~ 总时长）：
✓ 优先使用 position="结尾" 的片段
✓ 可以使用 position="中间" 的片段（如果语义合适）
✗ 避免使用 position="开头" 的片段

示例（假设总时长80秒）：
后16秒（64-80s）的片段应该是：
- seg_007 (position="结尾", type="活动福利") ✓
- seg_008 (position="结尾", type="行动号召") ✓

可接受：
- seg_006 (position="中间", type="价格对比") ✓ 可以放结尾

错误示例：
- seg_001 (position="开头", type="噱头引入") ✗ 不建议放结尾
```

**检查3：中间片段的灵活性**
```
新视频的中间60%位置（{总时长*0.2} ~ {总时长*0.8}）：
✓ position="中间" 的片段可以自由排列
✓ 只需保证语义连贯性
✓ 可以根据策略调整顺序
```

#### 自动修复策略

如果发现位置违规，使用以下修复策略：

```
违规1：结尾片段出现在开头
修复：寻找相同类型但 position="开头" 或 "中间" 的替代片段

违规2：开头片段出现在结尾
修复：寻找相同类型但 position="结尾" 或 "中间" 的替代片段

违规3：无法找到替代片段
修复：打破位置约束，但必须在 reasoning 中说明理由
      例如："虽然seg_008是结尾片段，但因为没有其他'行动号召'
            类型的开头片段，且语义流畅，所以破例使用"
```

**决策输出（步骤4）**：
```
位置约束检查结果：
开头区域（0-16s）：
  - seg_001 (开头) ✓
  - seg_002 (开头) ✓

中间区域（16-64s）：
  - seg_003 (中间) ✓
  - seg_004 (中间) ✓
  - seg_005 (中间) ✓
  - seg_006 (中间) ✓

结尾区域（64-80s）：
  - seg_007 (结尾) ✓
  - seg_008 (结尾) ✓

总体：✓ 全部通过

### 步骤5️⃣：差异化检查（与已有方案对比）

如果这是第N个方案（N > 1），需要与已生成的方案进行差异化检查。

#### 差异化维度

**维度1：策略差异（最重要）**
```
检查：本方案的核心策略是否与已有方案重复？

已有方案策略列表：
- 方案001: 深度种草型
- 方案002: 快节奏冲击型
- 方案003: 痛点堆叠型
...

本方案策略：{当前策略}

判断：
✓ 如果策略不同 → 通过
✗ 如果策略相同 → 需要调整策略或改变实现方式
```

**维度2：开场方式差异**
```
检查：本方案的开场方式是否与已有方案不同？

已有方案的开场方式：
- 方案001: 痛点描述开场
- 方案002: 多问题快速列举
- 方案003: 细节场景描述
...

本方案开场：{seg_001的类型和内容}

判断：
✓ 如果开场方式明显不同 → 通过
△ 如果有相似但有差异 → 可接受
✗ 如果完全相同 → 必须更换开场片段
```

**维度3：时长差异**
```
检查：本方案的时长是否与已有方案形成分布？

已有方案的时长分布：
- 71-73s: 2个方案
- 75-77s: 4个方案
- 85-90s: 2个方案

本方案时长：{total_duration}

建议：
✓ 选择未被充分覆盖的时长区间
✓ 避免时长"聚堆"现象
```

**维度4：片段组合差异**
```
检查：本方案使用的片段组合是否有独特性？

片段重复率计算：
重复率 = (与某个已有方案共同使用的片段数) / 本方案总片段数

判断标准：
✓ 重复率 < 40% → 差异明显，通过
△ 重复率 40-60% → 有差异，但需检查顺序是否不同
✗ 重复率 > 60% → 差异不足，需调整
```

**维度5：受众定位差异**
```
检查：本方案的目标受众是否有独特性？

已有方案的受众定位：
- 方案001: 理性决策型
- 方案002: 冲动决策型
- 方案003: 头皮问题严重人群
...

本方案受众：{target_audience}

判断：
✓ 如果受众不同 → 通过
△ 如果受众重叠但策略不同 → 可接受
```

#### 差异化得分计算

```
差异化总分 = 策略差异(30分) + 开场差异(20分) +
             时长差异(15分) + 片段组合差异(20分) +
             受众差异(15分)

评级：
90-100分：差异化优秀，强烈推荐
75-89分： 差异化良好，可接受
60-74分： 差异化一般，需优化
< 60分：  差异化不足，必须重新设计
```

```
差异化检查结果：
- 策略差异：✓ 本方案是"反套路型"，已有方案中无此策略 (30/30)
- 开场差异：✓ 反向问句开场，与已有8种开场都不同 (20/20)
- 时长差异：✓ 77秒，填补了75-80秒的空白区间 (15/15)
- 片段组合：✓ 与方案002的重复率35%，可接受 (18/20)
- 受众差异：✓ 针对"广告免疫的年轻人"，独特定位 (15/15)

总分：98/100 → 差异化优秀 ✓
```

### 步骤6️⃣：输出完整方案JSON

#### 字段填写要求

**scheme_id**
- 格式：scheme_001, scheme_002, ..., scheme_010
- 三位数字，前导零

**name**（方案名称）
- 要求：简洁、有力、易记
- 格式：{策略类型}-{核心特点}
- 示例：
  - "深度种草-完整攻略"
  - "快节奏多源混剪"
  - "价格爆破-超值感"
  - "反套路-真实体验"
  - "秋冬场景-应季种草"

**style**（广告风格）
- 必须从 ad_styles 中选择一个明确的风格
- 或者创建新风格（但需说明特征）

**description**（方案描述）
- 一句话总结方案的核心策略
- 示例：
  - "完整信息路径，充分建立信任后转化"
  - "快速节奏，多源素材密集轮转"
  - "反向营销，破除广告免疫心理"

**target_audience**（目标受众）
- 精准定义受众特征
- 示例：
  - "理性决策型消费者"
  - "冲动型购买人群"
  - "秋冬头皮问题用户"
  - "广告免疫的年轻人"
  - "价格敏感型用户"

**narrative_structure**（叙事结构）
- 用箭头连接片段类型，展示逻辑流
- 示例：
  - "痛点→方案→效果→价格→福利→CTA"
  - "噱头→痛点→方案→效果→信任→CTA"
  - "价格→方案→福利→紧迫CTA"

**reasoning**（片段选择理由）
- 每个片段都要说明：
  1. 为什么选择这个片段？（数据支撑、内容质量、功能价值）
  2. 它在方案中的作用是什么？（吸引、共鸣、说服、转化）
- 示例：
  - "质量评分9.0，开场点击率高，排比结构有冲击力，快速建立品牌关联"
  - "详细的成分讲解，建立专业性，适合理性决策用户"
  - "多重福利堆叠，强化超值感，制造购买紧迫性"

**position_reasoning**（位置选择理由）
- 说明为什么这个片段放在这个位置
- 考虑：
  1. 是否遵守位置约束？
  2. 与前后片段的逻辑关系？
  3. 在情绪曲线中的作用？
- 示例：
  - "开场必须快速吸引，噱头引入最适合"
  - "在产品方案后展示效果，形成'功能→结果'的自然逻辑"
  - "结尾需要行动号召，制造紧迫感驱动转化"

**strategy_reasoning**（整体策略理由）
- 总结整个方案的设计思路
- 说明为什么这样组合能达到目标
- 示例：
  - "针对理性用户，采用完整信息路径，从痛点到方案到效果到信任，
     每一步都有充分的论证，最后用福利和CTA自然转化"
  - "针对信息流场景，采用快节奏切换，每5-8秒一个片段，
     密集的信息轰炸，快速建立印象并转化"

**differentiation**（差异化说明）
- 说明与已有方案的差异
- 示例：
  - "与方案001的深度种草不同，本方案采用反套路策略，
     先承认缺点再展示真实效果，建立独特的信任感"
  - "与方案002的快节奏不同，本方案聚焦价格优势，
     开场直接展示低价，然后堆叠福利，纯粹的价格驱动"

## 常见错误及纠正

### ❌ 错误1：位置约束违规

**错误示例**：
```json
{
  "segments": [
    {
      "position": 1,
      "segment_id": "seg_008",
      "type": "行动号召",
      "segment_position": "结尾"  // ✗ 结尾片段不能放在开头
    }
  ]
}
```

**纠正方法**：
1. 寻找相同类型但 position="开头" 的片段替代
2. 或者选择不同类型但功能相似的开头片段
3. 如果必须使用，在 reasoning 中说明理由

---

### ❌ 错误2：语义跳跃

**错误示例**：
```json
{
  "segments": [
    {"position": 1, "type": "产品方案"},
    {"position": 2, "type": "行动号召"},  // ✗ 太快了
    {"position": 3, "type": "痛点"}       // ✗ 逻辑倒置
  ]
}
```

**纠正方法**：
- 遵循自然的逻辑流：吸引 → 共鸣 → 说服 → 转化
- 检查相邻片段的类型组合是否在"允许列表"中

---

### ❌ 错误3：差异化不足

**错误示例**：
```
方案002: 快节奏冲击型，77.5秒，7段
方案004: 快节奏冲击型，77.2秒，7段  // ✗ 与方案002几乎相同
```

**纠正方法**：
- 改变核心策略（如改为"价格爆破型"）
- 改变开场方式（如从"噱头"改为"价格"）
- 调整时长（如改为85秒，节奏放慢）

---

### ❌ 错误4：片段选择缺乏理由

**错误示例**：
```json
{
  "reasoning": "这个片段不错",  // ✗ 太空泛
  "position_reasoning": "放这里合适"  // ✗ 没说为什么
}
```

**纠正方法**：
```json
{
  "reasoning": "质量评分9.0，痛点描述具象化（'油得像条形码'），
                引发强烈共鸣，适合开场建立问题认知",
  "position_reasoning": "开场第一个片段，快速触达用户痛点，
                         建立'我就是你'的共鸣感"
}
```

---

### ❌ 错误5：时长控制失败

**错误示例**：
```
目标时长：50-110秒
实际时长：45.3秒  // ✗ 低于最小值
```

**纠正方法**：
- 增加片段数量
- 选择时长较长的片段
- 检查是否遗漏了某些必要类型（如：福利、CTA）

# 伪代码：计算两个方案的差异度
def calculate_difference(scheme_a, scheme_b):
    score = 0

    # 策略差异（30分）
    if scheme_a.strategy != scheme_b.strategy:
        score += 30

    # 开场差异（20分）
    if scheme_a.segments[0].id != scheme_b.segments[0].id:
        score += 20

    # 时长差异（15分）
    duration_diff = abs(scheme_a.duration - scheme_b.duration)
    score += min(duration_diff / 5, 15)  # 每5秒差异+1分，最多15分

    # 片段组合差异（20分）
    common_segments = len(set(scheme_a.segment_ids) & set(scheme_b.segment_ids))
    repetition_rate = common_segments / len(scheme_a.segment_ids)
    score += (1 - repetition_rate) * 20

    # 受众差异（15分）
    if scheme_a.target_audience != scheme_b.target_audience:
        score += 15

    return score

# 目标：任意两个方案的差异度 >= 75 分
```

## 总结：核心要点

### 三个必须遵守的原则
1. **位置约束 > 其他约束** - 绝不能把结尾片段放开头
2. **语义连贯 > 数据指标** - 宁可牺牲高分片段，也要保证逻辑流畅
3. **差异化 > 完美方案** - 10个不同的好方案 > 10个相似的完美方案

### 三个黄金比例
1. **时长分布**：50-70s (30%) | 70-90s (50%) | 90-110s (20%)
2. **片段数量**：5-7段 (30%) | 7-9段 (60%) | 9-10段 (10%)
3. **质量片段**：score >= 8.5 的片段占比 >= 60%

### 三个检查点
1. **位置约束检查**：开头20%无结尾片段，结尾20%无开头片段
2. **语义连贯检查**：相邻片段符合"允许组合"列表
3. **差异化检查**：与已有方案的差异度 >= 75分
